## 1. Search Algorithm

Your current search (`search.go`) only searches **users** (username, fullName, league) with fuzzy/Levenshtein matching. Your search **page** (`search_page.dart`) does client-side filtering on challenge title/creator. Two paths to improve:

**Option A — Extend backend search to include challenges**

- Score challenges by relevance (title match), recency, views, likes, response count
- Return mixed results: `{ users: [...], challenges: [...] }`

**Option B — Full-text search with PostgreSQL**
- Use PostgreSQL's built-in `tsvector` / `tsquery` full-text search
- Add a `search_vector` column to challenges and users tables, auto-updated via trigger
- Much faster and more accurate than loading all records into memory
- Supports ranking, stemming, partial matches out of the box

**Option C — External search engine (Meilisearch / Typesense / Algolia)**
- Deploy a lightweight search service (Meilisearch is free/self-hosted)
- Index users + challenges on create/update
- Instant, typo-tolerant search with facets and filters
- Best UX but adds infrastructure

**My recommendation:** Option B (PostgreSQL full-text search) — no extra infra, big upgrade over current approach, handles thousands of records easily.

## 2. Adding Music to Videos

**Option A — Pre-upload music selection (simplest)**
- User picks a song from your music library BEFORE recording

- Flutter packages: `just_audio` for playback, `ffmpeg_kit_flutter` to merge audio+video

**Option B — Post-upload music overlay**
