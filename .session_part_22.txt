- **Analysis is allowed** — explaining what code does is defensive; security researchers need that.
- **Augmentation is refused** — writing, editing, or extending the code means the AI is materially contributing to something potentially harmful.

The rule is applied as a blanket policy on every file read because:
1. The assistant can't always reliably tell malware from benign code. Obfuscated droppers can look like "a networking utility." Credential stealers can look like "a password manager."
2. Even if the assistant judges correctly 99% of the time, the 1% where it's wrong is the exact case the guardrail is protecting against.
3. A blanket rule is harder to social-engineer around than a judgment-based one. ("Trust me, it's for a CTF" / "it's my own code" / "it's for defense.")
