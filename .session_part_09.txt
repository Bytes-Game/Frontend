Great question. Here's why:

## Why Instagram & TikTok build custom video engines

### 1. Performance at scale
Off-the-shelf players (ExoPlayer, AVPlayer) are designed for **general use**. When you have **2 billion users** scrolling through videos, every millisecond matters:
- They need videos to start in **<100ms** (instant feel)
- Standard players take 300-500ms to initialize
- Custom engines **pre-decode the next 3-5 videos** before you even swipe

### 2. Memory control
A TikTok feed has potentially infinite videos. Standard players weren't designed for this:
- ExoPlayer creates a new instance per video → **memory leaks** with many videos
- Custom engines **reuse a pool of 2-3 decoder instances** across all videos
- They control exactly when to allocate/release GPU memory

### 3. Network optimization
Standard players just download and play. These apps need:
- **Adaptive bitrate** — start with low quality on slow networks, upgrade seamlessly
- **Prefetching** — download the first 2 seconds of the next 5 videos while you watch the current one
- **Custom caching** — keep recently viewed videos in memory so scrolling back is instant
- Standard players don't support this level of control

### 4. Codec control
They want to use **newer codecs** (AV1, VP9, HEVC) that save 30-50% bandwidth:
- Standard players depend on device OS support
- Custom engines **bundle their own decoders** so they work on every device
- Instagram saves **millions of dollars** in CDN costs by using AV1 where possible

### 5. Feed-specific UX
Standard players are built for "open video → watch → close." But reels/shorts need:
- Seamless **vertical swipe** transitions with no loading gap
- **Auto-play/auto-pause** based on visibility percentage
- **Loop detection** — track how many times a user rewatches (for the algorithm)
- Sound sync across swipes
- None of this exists in off-the-shelf players

### 6. Analytics & algorithm
The recommendation algorithm needs **granular playback data** that standard players don't expose:
- Exact frame where user lost interest and swiped away
- Replay count, pause points, skip-back points
- Network stall events correlated with user retention
- Custom engines emit **hundreds of custom events** per video session

|---|---|
| 2 billion users | Early stage |
| Millions in CDN costs to optimize | Standard hosting |
